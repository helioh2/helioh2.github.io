    • O código para a função custo está escrito abaixo:  

    • Até agora, todas as funções que você fez, fazem a mesma coisa com suas entradas:

      • triangulo-verde sempre faz triângulos verdes, não importa o tamanho recebido.

      • esquerda-salvo? sempre compara a coordenada de entrada com -50, não importando qual número seja.

      • atualizar-perigo sempre soma ou subtrai o mesmo valor.

      • e assim por diante...

      Isso ficou evidente quando se passa os Exemplos para a definição: circulando o que muda sempre resulta na definição, e o número de variáveis sempre coincidiu com o número de coisas no Domínio.

      Vá para a página Page 23, e preencha a Assinatura e Exemplos para esta função, e então circule e nomeie o que muda.

      Pode ser útil ter alguns Exemplos e definições escritos no quadro, para que os alunos possam acompanhar.

    • A função custo é especial, porque diferentes sabores resultam em diferentes expressões para calcular. Se você tivesse que circular tudo o que muda no exemplo, você teria que circular os sabores e o preço. Essas são duas coisas que mudam, mas o Domínio da função tem somente um item - por isso, não podemos ter duas variáveis.

    • Claro que, preço não é uma variável independente, já que o preço depende interiamente do sabor. Por exemplo: se o sabor é "queijo" a função irá simplismente produzir 9.00, se o sabor for calabresa a função irá produzir 10.50, e asssim por diante. O preço ainda é definido em função do sabor, e existem quatro possíveis sabores - quatro condições possíveis - que a função deve se preocupar. A função custo faz uso de um recurso especial da linguagem chamado condicionais , que é abreviado no código como cond.

    • Cada condicional tem pelo menos uma condição. Cada condição tem uma pergunta Booleana e um resultado. Na função do Luigi, existe uma condição para queijo, outra para calabresa, e etc. Se a condição é verdadeira, a expressão é calculada e retorna um valor. Se a condição é falsa, o computador pula para a próxima condição.

      Dê uma olhada na função custo:

      • Quantas condições existem para a função custo?

      • Identifique a pergunta da primeira condição.

      • Identifique a pergunta da segunda condição.

      Colchetes cercam a pergunta e a resposta em cada condição. Quando os alunos identificam as questões, eles devem encontrar a primeira expressão dentro dos colchetes. Existe apenas uma expressão para cada resposta.

    • A última condição em uma condicional pode ser um else (senão), que será executado se todas as perguntas forem falsas.

      Na função custo, o que é retornado se todas as condições forem falsas? O que aconteceria se não existisse a condição else? Tente remover ela do código e calcule um sabor desconhecido, e veja o que acontece.

      O else é muito usado como um "pega-tudo" para casos que não podem ser numerados. Se você pode indicar uma pergunta precisa para uma condição, escreva essa pergunta precisa ao invés de else. Por exemplo, se você tem uma função que faz coisas diferentes dependendo se alguma variável x é maior do que 5, é melhor para iniciantes escrever duas condições (> x 5) e (<= x 5) em vez de ter a segunda condição sendo else. Questões explícitas tornam os programas fáceis de ler e fazer manutenção. Quando você usa else, alguém tem que ler todas as condições anteriores para saber com qual condição o else se corresponde: eles não podem voltar e fuçar nas condições anteriores para encontrar uma que combina com alguma situação. Isso pode ser um pouco contra-intuitivo para aqueles com alguma experiência anterior em programação, mas ajuda a tornar o código mais legível e compreensível.

    • Funções que usam condicionais são chamadas funções segmentadas, porque cada condição define uma parte separada da função. Porque funções segmentadas são úteis? Pense sobre o personagem principal em seu jogo: você gostaria que ele se movesse de uma maneira se você aperta a seta "para cima", e de outra maneira de você apertar a tecla "para baixo". Se mover para cima e se mover para baixo precisam de duas funções diferentes! Sem o cond, você poderia apenas escrever uma função que sempre move o personagem para cima, ou sempre o mova para baixo, mas não os dois juntos.

    Movimento do Personagem

    Overview

    Alunos escrevem uma função segmentada que permite o personagem se mover para cima e para baixo usando as setas.

    Learning Objectives

    • Os alunos aprendem que a manipulação de entradas de usuário precisam de funções segmentadas

    • Os alunos aprendem quais perguntas devem ser feitas para detectar teclas pressionadas

    • Os alunos aprendem como escrever suas próprias expressões condicionais

    • Os alunos raciocinam sobre o posicionamento dos objetos usando matemática

    Evidence Statements

    • Os alunos irão aprender como escrever expressões diferentes que calculam novas coordenadas em diferentes direções

    • Os alunos serão capazes de escrever expressões que verificam qual tecla foi pressionada

    • Os alunos serão capazes de escrever uma condional de ligação de teclas pressionadas para diferentes direções de movimento

    • Os alunos irão aprender a escrever exemplos que ilustram cada condição em uma função segmentada

    • Os alunos serão capazes de experimentar com o uso de funções para mudar a velocidade ou a natureza do movimento do personagem em um jogo

      Movimento do Personagem (Time 25 min)

      • Agora que sabemos do cond, podemos escrever atualizar-jogador.

        Olhe a seguinte figura, e descreva o que acontece quando a tecla seta "para cima" é pressionada.

        • Com qual coordenada x o jogador começa?

        • Com qual coordenada y o jogador começa?

        • E depois que o jogador se mover?

          • Quais são as novas coordenadas x e y?

          • O que mudou e quanto mudou?

          • O que acontece quando apertamos a tecla seta "para baixo"?

          • Quais devem ser as novas coordenadas?

          • O que deve acontecer se o usuário pressionar alguma outra tecla que não é seta "para cima" ou seta "para baixo"?

        Desenhe uma tela no quadro, e coloque coordenadas para o personagem, alvo e perigo. Circule todos os dados associados com o personagem Jogador.

      • A tabela a seguir resume o que deve acontecer com o jogador quando cada tecla é pressionada:

        Quando...

        Faça...

        tecla é "up(para cima)"

        adicione 20 para jogador-y

        tecla é "down(para baixo)"

        diminua 20 de jogador-y

        tecla é qualquer outra

        retorne y sem alteração

        Na Page 24 de sua apostila, você encontrará um problema para atualizar-jogador.

        (Se você não gostar de usar as setas de direção para fazer o jogador mover-se para cima e baixo, você pode facilmente mudá-los para trabalhar com "w" e "s"!)

        Certifique-se de verificar os EXAMPLEs e Assinaturas dos alunos durante esse exercício, especialmente quando for hora de circular e nomear o que é variável entre os exemplos. Este é um passo fundamental na Receita de Projeto onde eles devem descobrir a necessidade de usar o cond.

      • Você também pode adicionar movimentos mais avançados, usando o que você aprendeu sobre funções booleanas. Aqui estão algumas ideias:

        • Teletransporte: ao invés de ter a coordenada y somada ou diminuída, substitua ela com um número para que o o jogador apareça de repente nessa localização. (Por exemplo, pressionando a tecla "c" faz o jogador voltar ao centro da tela, em y=240.)

        • Detecção de Limites Altera a condição de se movimentar para cima para que o jogador só consiga subir se a tecla="up" AND jogador-y for menor que 480. Da mesma maneira, altera a condição para baixo para verificar se jogador-y é maior do que 0.

        • Portal: Adiciona uma condição (antes de qualquer tecla) que verifica se a coordenada-y do jogar está acima da tela (y > 480). Se estiver, faça o jogador aparecer na parte de baixo(y=0). Adicione outra condição que o jogador apareca no topo da tela se ele se mover abaixo da parte inferior da tela.

        • Desafio: Faça o jogador ficar invisível quando a tecla "h" for pressionada, só re-apareça quando ela for pressionada novamente!

        Dica para o Desafio: multiplique por -1.
